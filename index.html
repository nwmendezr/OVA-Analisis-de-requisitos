ANALISIS DE REQUISITOS

El curso Análisis de Requisitos hace parte del componente disciplinar del programa Tecnología en desarrollo de Software. Se centra en el estudio de la disciplina de la Ingeniería de Requisitos, la cual permite mejorar el entendimiento, análisis, validación, especificación y gestión de las necesidades, funcionalidades y restricciones que debe cumplir un sistema-software, El curso se encuentra estructurado con las siguientes unidades.

UNIDAD UNO DESARROLLO DE SOFTWARE Y ANÁLISIS DE PROBLEMAS

1. Generalidades y conceptos de Ingeniería del software

2. Análisis de problemas

3. El proceso de software

4. Técnicas para el levantamiento de información

5. Modelos y Métodos de Negociación de Requerimientos

UNIDAD DOS INGENIERÍA DE REQUISITOS (IR)


1. Ingeniería de requerimientos

2. Levantamiento de requerimientos

3. Análisis de requerimientos

4. Modelo de Requisitos

5. Especificación de requisitos

UNIDAD TRES ESTÁNDARES PARA LA ESPECIFICACIÓN DE REQUERIMIENTOS

1. Estándares y modelos formales para especificación de requerimientos

2. Documento de requisitos

3. Caso de estudio- Ejemplo práctico

4. Requisitos y Calidad del software

OBJETIVOS

* Brindar al estudiante las bases conceptuales y metodológicas necesarias para que realice el levantamiento de la información sobre un problema. utilice las técnicas para recolectar la información de requerimientos funcionales y no funcionales así como la especificación de los mismos, teniendo en cuenta las tendencias actuales en la industria de desarrollo de software.
* Fundamentar, en el estudiante, la importancia de la ingeniería de requisitos a través de los principios de desarrollo del software, alcances y características, utilizando estrategias de aprendizaje propias de la educación a distancia, para garantizar el completo desarrollo y aprehensión de la temática planteada.
